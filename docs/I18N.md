# PROOF · Localización (i18n)

Fundación **L0** del epic [#13](https://github.com/paunrv/fermentrack/issues/13).  
Stack: [next-intl](https://next-intl.dev) + App Router (Next.js 14).

## Locales

| Código   | Rol     |
|----------|---------|
| `es-MX`  | Default |
| `en-US`  | Inglés  |

Persistencia: cookie `NEXT_LOCALE` (1 año). Sin cookie, el middleware usa `Accept-Language`.

Rutas **sin prefijo** (`localePrefix: 'never'`) — `/dashboard/...` no cambia.

## Archivos clave

| Ruta | Propósito |
|------|-----------|
| `apps/web/src/i18n/routing.ts` | Locales + cookie |
| `apps/web/src/i18n/request.ts` | Carga de mensajes por request |
| `apps/web/src/i18n/navigation.ts` | `Link` / `useRouter` locale-aware |
| `apps/web/messages/es-MX.json` | Strings es-MX |
| `apps/web/messages/en-US.json` | Strings en-US |
| `apps/web/src/lib/i18n/format.ts` | Fecha, número, MXN |

## Convención de keys

- Namespace de primer nivel = área de producto: `common`, `landing`, `about`, `contact`, `legal`, `auth`, `profileSelect`, `onboarding`, `dashboard`, …
- Keys en **camelCase**, anidadas por pantalla o componente.
- Interpolación ICU: `{variable}` — ver [next-intl rich text](https://next-intl.dev/docs/usage/messages).
- No hardcodear copy en JSX salvo prototipos; migrar en issues L1–L8.

Ejemplo:

```json
{
  "winemaker": {
    "lots": {
      "title": "Lotes",
      "empty": "Aún no hay lotes"
    }
  }
}
```

Uso en Server Component:

```tsx
import { getTranslations } from 'next-intl/server'

const t = await getTranslations('winemaker.lots')
return <h1>{t('title')}</h1>
```

Uso en Client Component:

```tsx
'use client'
import { useTranslations } from 'next-intl'

const t = useTranslations('winemaker.lots')
```

## Formatos

```tsx
import { formatDate, formatCurrencyMxn } from '@/lib/i18n/format'
import { useLocale } from 'next-intl'

const locale = useLocale() // 'es-MX' | 'en-US'
formatDate(new Date(), locale)
formatCurrencyMxn(1299, locale)
```
